# Sky Logistics / 天穹物流

**其他语言版本： [English](README.md)**

为 Minecraft 提供天穹主题的无线物流系统。天穹物流通过命名线路搬运物品、流体和能量，提供聚合型大型仓储，并包含用于配置网络、过滤、随身转运和高空供奉配方的工具链。

## 功能亮点

- **天生无线**：通过命名物流线路连接机器、仓库和接口，无需铺设实体管道；物品、流体和能量共用同一套线路模型，并可通过维度升级跨维度传输。
- **高吞吐量**：普通节点可处理 1 堆叠/t，安装速度升级后提升至 2 堆叠/t；物品与能量单次传输默认采用 2.1G 级上限，模组内仓储直连默认采用 9E 级上限。
- **面向服务器性能**：节点传输使用操作预算、就绪线路队列、热槽追踪、能力缓存和端点退避，减少无效扫描，让大型网络也能保持响应。
- **快速铺设与配置**：物流节点可根据是否潜行快速切换放置模式；天穹配置器支持线路管理、节点配置复制粘贴和副手放置预设。
- **自带高堆叠仓储**：天穹物品库和天穹流体库按类型聚合存储，单类型可堆叠至 9E 级别，并提供可搜索的终端式界面。
- **物品栏与背包交互**：天穹项链可在物流线路、玩家物品栏和支持的背包库存之间转运物品，方便随身补货或卸货。
- **模组联动**：与 Jade、JEI、Patchouli、Curios、Sophisticated Backpacks、Mekanism、1.20.1 植物魔法和新生魔艺等可选兼容，并可为 AE2、Refined Storage、Beyond Dimensions 网络提供 G/t 级吞吐连接。

## 安装

1. 将与你的 Minecraft/加载器版本匹配的天穹物流 jar 放入 `mods` 文件夹
2. 按需安装想要使用的可选联动 Mod
3. 启动游戏

## 使用方法

1. 在配置的高空供奉高度为尤洛伽水晶充能，并用它制作天穹石和天穹物流组件
2. 搭建带供桌的天穹供奉祭坛，用于制作柯拉甘露和其他供奉产物
3. 放置天穹物品库或天穹流体库，作为聚合仓储端点
4. 将天穹物流节点放在机器、仓库或接口旁；普通放置为存入模式，潜行放置为抽取模式
5. 使用天穹配置器创建/选择线路、重命名线路、复制节点设置、粘贴节点设置，并通过副手为新放置节点预设配置
6. 按需给节点加入天穹过滤列表、速度升级卡和维度升级卡
7. 给天穹项链放入白名单过滤列表，即可在玩家物品栏与物流线路之间随身抽取或存入物品

## 说明

- 天穹物流不是逐格连接的实体管道网络，而是直接配对同一线路中已加载的抽取面和存入面。
- 线路没有隐藏的物品/流体/能量缓存。目标无法接收时，源端不会先被抽空。
- Mekanism 化学品走流体开关；植物魔法 mana 和新生魔艺 Source 魔源走能量开关，但只会搬到同类资源处理器，不会转换为 FE。
- 显示名对应的线路 id 是稳定的；不改名或复用同名线路时，会继续指向同一条线路。
- 节点传输使用预算、缓存、就绪线路队列、热槽追踪、能力缓存和端点退避来控制性能开销。
- 天穹项链的工作间隔可通过服务端配置 `skyNecklaceTickInterval` 调整，默认 10 tick。
- 仓库类型上限、节点物品/能量单次传输量、天穹容器间直连传输量、操作预算、热槽缓存大小、供奉高度和水晶充能时间均可配置。
- Patchouli 支持为纯数据内容，安装 Patchouli 后会显示指南书内容。
- 可选联动仅在检测到对应 Mod 且版本/API 兼容时启用。

## 构建

在仓库根目录构建两个版本：

```bash
./scripts/build_all_versions.sh
```

单独构建一个版本：

```bash
cd versions/1.21.1
./gradlew --no-daemon clean build

cd ../1.20.1
./gradlew --no-daemon clean build
```

`common` 目录存放所有支持版本共享的源码和资源。Forge、NeoForge、映射、依赖和 API 相关代码保留在 `versions/<minecraft-version>` 下。
