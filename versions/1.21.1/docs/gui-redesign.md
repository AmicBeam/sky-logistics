# Sky Logistics GUI Redesign

本文是 `0.0.1` 之后的 GUI 重设计规格。当前状态：下方内容是目标设计与验收稿，不代表所有界面已经进入代码实现。

已进入代码的部分：

- 物流节点已有 4 个升级槽。
- 过滤列表已改为可见名称“天穹过滤升级”，并保留独立 GUI。
- 速度升级卡已控制节点速率：`1 堆叠/t -> 2 堆叠/t`。

尚未完整进入代码的部分：

- 配置器的线路翻页、线路摘要、空线路删除。
- 物流节点的 XNet 式每面物品/流体独立配置。
- Create 风格过滤升级新版布局。
- AE2/RS 风格物品库与流体库终端布局、排序、搜索和滚动网格。

## 参考模组

- [McJtyMods/XNet](https://github.com/McJtyMods/XNet)
  - 借鉴点：一个控制界面内配置多类资源通道；连接器按方向暴露配置，而不是让玩家逐个方块反复右键。
  - 对天穹物流的转化：物流节点 GUI 使用“六面矩阵”，每个面独立选择物品/流体开关与抽取/存入模式；调度索引按资源类型拆分。
- [Creators-of-Create/Create](https://github.com/Creators-of-Create/Create)
  - 借鉴点：Create 将普通列表过滤器与属性过滤器拆成清晰的两套交互：列表过滤器配置物品样本，属性过滤器配置物品属性。
  - 对天穹物流的转化：过滤升级保留为一个物品，但内部提供“列表模式/属性模式”切换；列表模式配置幽灵物品，属性模式配置属性芯片。
- [Applied-Energistics-2](https://github.com/AppliedEnergistics/Applied-Energistics-2)
  - 本地参考：`Applied-Energistics-2/src/main/java/appeng/client/gui/me/common/MEStorageScreen.java`
  - 借鉴点：终端核心区域是资源网格；左/侧工具栏放排序、视图、类型过滤；搜索框驱动 view 更新；滚动条与资源仓库解耦。
  - 对天穹物流的转化：物品库/流体库改为“终端式资源视图 + 玩家背包 + 快速存入取出交互”。
- [refinedmods/refinedstorage](https://github.com/refinedmods/refinedstorage)
  - 本地参考：
    - `refinedstorage2/refinedstorage-resource-api/src/main/java/com/refinedmods/refinedstorage/api/resource/repository/ResourceRepositoryImpl.java`
    - `refinedstorage2/refinedstorage-common/src/main/java/com/refinedmods/refinedstorage/common/support/widget/SearchFieldWidget.java`
    - `refinedstorage-main/src/main/java/com/refinedmods/refinedstorage/blockentity/grid/GridBlockEntity.java`
  - 借鉴点：资源列表分为 backing list 与 view list，排序/筛选作用于 view list；搜索框支持历史、自动聚焦；网格设置变化后触发重新排序。
  - 对天穹物流的转化：物品库/流体库使用缓存视图列表，排序和搜索不直接遍历后端存储；按钮状态变化必须触发局部刷新。
- [Direwolf20-MC/LaserIO](https://github.com/Direwolf20-MC/LaserIO)
  - 借鉴点：一个节点作为中心，多个面用小型视觉连接与状态色表达；升级槽、过滤、传输配置分区清楚。
  - 对天穹物流的转化：物流节点 GUI 不画实体管道，而是把六个面做成可扫描的设备卡片和颜色状态。

## 统一交互原则

- 图标优先，文字保留在标题、状态和值上；节点面模式不再写“无/抽取/存入”三段文字，而用三态按钮色块与 tooltip。
- 所有可切换按钮都要有即时视觉反馈：客户端先更新本地展示状态，服务端确认后同步真实状态。
- `粘贴模式` 不在配置器 GUI 中显示 ON/OFF，因为打开配置器 GUI 会退出粘贴模式。
- 线路列表默认只显示玩家自己的线路；管理员或后续共享功能再考虑显示公共线路。
- 删除线路只允许删除空线路：没有节点、没有外部设备接入、没有缓存的线路才能删除。
- 所有图标按钮必须有 tooltip；按钮可使用符号，但不要用大段文字解释操作。

## 天穹配置器

### 推荐布局：宽屏线路面板

- 尺寸建议：`360 x 232`，比当前 `224 x 144` 更宽，给线路摘要留空间。
- 顶部：标题 + 当前线路名/短 ID + 线路导航按钮，按钮紧跟在线路文字右侧。
- 线路导航按钮：
  - `|<` 返回玩家第一条线路
  - `<` 上一条线路
  - `>` 下一条线路；在最后一条时等价于创建新线路
  - `>|` 跳到最后一条线路
  - `×` 删除当前空线路
- 中部：线路状态区。
  - 显示当前线路已连接节点数。
  - 列出节点位置、面、接入设备图标、设备名称、资源类型摘要。
  - 设备列表可滚动；只展示已加载/已索引节点，缺失节点以灰色状态显示。
- 下部：操作区。
  - 复选框：物品、流体。点击后立即改变按钮填充状态。
  - 按钮：进入粘贴模式；不显示 ON/OFF，因为打开配置器 GUI 会退出粘贴模式。

### 必要数据模型

- `LineSummary`
  - owner UUID
  - line UUID
  - display index
  - node count
  - connected device summaries
  - empty/deletable flag
- 配置器打开时请求玩家可见线路列表；线路变更后只同步当前页摘要。
- 按钮点击刷新检查：
  - 当前代码中配置器按钮是静态 label，物品/流体状态通过 `renderLabels` 读取物品 NBT，点击后依赖服务端包写回物品并同步到客户端；这不是最终方案。
  - 新版应在 `ConfiguratorMenu` 增加同步数据或 `ClientboundConfiguratorStatePacket`，按钮点击后客户端先乐观更新，服务端确认后修正。
  - 线路摘要和当前线路页码不能从客户端物品 NBT 临时拼；必须由服务端线路索引返回。

## 天穹物流节点

### 推荐布局：LaserIO 风格六面标签页 + 面详情

- 尺寸建议：`384 x 292`
- 顶部同配置器线路选择区：线路文字右侧放置 `|<`、`<`、`>+`、`>|`、`x` 紧凑按钮。
- 顶部第二行：6 个面标签按钮，类似 LaserIO 节点卡片切页。
  - 每个按钮显示相邻方块/设备的物品图标、面的单字母/汉字简写和模式色条。
  - 完整方向名与设备名显示在当前面详情标题和 tooltip 中，避免面切换区退化成文字列表。
  - 有接入设备的面可点击切换页面。
  - 无设备的面置灰，不可点击。
  - 当前面使用高亮描边与填充色。
- 中部：当前所选面的详细配置。
  - 当前面、资源开关、模式选择必须压成紧凑配置区，避免在 254 像素宽的真实 GUI 里形成大面积空白。
  - 当前设备名与方向，不显示“物品能力/流体不可用/能量 不可用”这类状态标签。
  - 该面的物品/流体独立开关。
  - 模式选择为三个文字按钮：无、抽取、存入；当前选中项同时用填充色表达。
  - 不显示过滤格子。过滤规则只来自玩家预先配置好的“过滤升级”物品。
  - 不显示过滤升级摘要。升级槽内的过滤升级只通过物品图标表达。
  - 不显示节点传输速率；速率仅由实际升级效果决定。
- 底部：玩家背包。
- 当前面详情右侧/底部：4 个升级槽显示短标签、槽位和物品图标。
  - 视觉上允许保留一个短标签“升级插槽”，但不显示过滤摘要、速率、说明句或其他长文本。
  - 允许放入过滤升级、速度升级卡、维度升级卡。
  - 每个种类同一节点只能插入 1 张。
  - 过滤升级是唯一可以自己打开 GUI 配置的升级物品。
  - 速度升级卡将节点速率从 `1 堆叠/t` 提升至 `2 堆叠/t`。
  - 维度升级卡先作为跨维度线路能力的预留升级；当前公开测试版不启用跨维度调度。

### 每面配置模型

现有节点配置需要从“节点全局物品/流体开关”升级为“每面开关”：

- `FaceConfig`
  - `NodeFaceMode mode`
  - `boolean itemsEnabled`
  - `boolean fluidsEnabled`
  - cached item capability availability flag
  - cached fluid capability availability flag

过滤与速率不放在 `FaceConfig` 中，避免同一节点六个面重复配置，也避免节点 GUI 内部再出现过滤格子。节点本体持有 4 个升级槽：

- 过滤升级：提供白/黑名单与 tag/精确匹配。
- 速度升级卡：节点扫描/传输速率从 `1 堆叠/t` 提升至 `2 堆叠/t`。
- 维度升级卡：后续跨维度线路的能力开关。

调度器索引也应从 `line -> inputs/outputs` 改为按资源类型维护：

- `line -> itemInputs / itemOutputs`
- `line -> fluidInputs / fluidOutputs`

这样禁用流体的面不会进入流体热路径，禁用物品的面不会进入物品热路径。

## 天穹过滤升级

可见名称改为“天穹过滤升级”；注册 ID 暂保留 `filter_list`，用于兼容 `0.0.1` 测试存档和已有物品。

### 推荐布局：Create 风格双模式过滤卡片

- 尺寸建议：`176 x 166`
- 上方：标题和三个小按钮。
  - 列表/属性模式切换
  - 白/黑名单
  - 列表模式下为精确/tag；属性模式下为任一/全部
- 列表模式中部：`3 x 3` 大过滤槽，居中留白，像一个实际的物品卡片。
- 属性模式中部：属性芯片，不显示物品格。
  - 可堆叠
  - 不可堆叠
  - 有 NBT
  - 可损坏
  - 已损坏
  - 附魔
  - 食物
  - 方块物品
- 下方：玩家背包。
- 交互：
  - 左键槽：放入当前光标物品的幽灵项。
  - 右键槽：清空该槽。
  - Shift 点击玩家物品：填入第一个空过滤槽。
  - 属性模式点击芯片切换属性是否启用。
  - 白名单表示“匹配时允许”，黑名单表示“匹配时拒绝”。

## 天穹物品库 / 天穹流体库

### 推荐布局：AE2/RS 终端式

- 尺寸建议：`296 x 256` 起，支持根据屏幕高度扩展资源行数。
- 顶部：
  - 搜索框
  - 排序按钮：数量、名称、模组、最近变化
  - 方向按钮：升序/降序
  - 过滤按钮：全部/非空、物品/流体类型
- 中部：
  - 标准资源网格区域，使用 18x18 格子。
  - 每格显示图标、堆叠数量缩写、hover 详情。
  - 自动按 view list 排序；资源变化时增量更新。
- 底部：
  - 玩家背包与快捷栏。
  - 玩家可以 shift 点击或拖拽物品存入容器。
  - 点击终端格：左键取一组，右键取半组或一件，Shift 点击取至背包可容纳数量。
- 左侧窄工具栏：
  - 存入背包全部同类
  - 清空搜索
  - 锁定排序
  - 切换显示模式：普通网格、紧凑数量、仅可取出

### 视图与排序策略

采用 RS 风格的资源仓库思想：

- 后端 `StoredResourceMap` 保存真实数量。
- 前端 `ResourceViewList` 保存当前搜索和排序后的条目。
- 搜索变更：重新计算 view list。
- 数量变更：只更新受影响条目；如果排序锁定，延迟重排，避免高频变化造成 UI 抖动。
- 默认排序：数量降序，其次名称，其次注册 ID，保证稳定排序。

## 当前按钮刷新问题检查

`0.0.1` 中：

- 过滤板按钮已通过 `containerTick` 刷新显示文本。
- 配置器当前仍通过 `renderLabels` 显示物品/流体 ON/OFF 与粘贴模式 ON/OFF；这不符合新版设计，粘贴模式状态应从 GUI 移除。
- 节点 UI 目前仍主要读取客户端方块实体状态；服务端更新后依赖 block entity update packet。

下一版实现要求：

- 配置器和节点 GUI 都引入专用 `MenuData` 或轻量同步包，按钮点击后客户端先乐观更新，服务端确认后修正。
- 线路列表、面 capability 状态、节点接入设备列表不从客户端方块实体临时推断，而由服务端索引摘要下发。
- 所有按钮的 label 和色块都在 `containerTick` 或状态包回调中重算，避免“点了按钮但 GUI 没变”的旧问题。
- 终端搜索框输入、排序按钮、非空过滤按钮都必须只重算 view list，不直接改后端存储。

## 推荐取舍

- 配置器选 A：线路信息需要显示“当前已连线节点和设备”，紧凑版会太挤。
- 物流节点选 A：XNet 类配置最适合矩阵，后续扩展能量/Mek 气体也不挤。
- 过滤器选 A：更像 Create 的过滤物品，玩家学习成本最低。
- 物品/流体库选 A：这是核心容器体验，应该优先做到 AE2/RS 风格。
